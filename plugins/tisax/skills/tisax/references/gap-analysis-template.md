# TISAX Gap Analysis Template
## Structured Assessment Template Against VDA ISA 6.0

---

## How to Use This Template

1. Download the full VDA ISA 6.0 questionnaire from https://www.vda.de
2. Use this template to structure your internal gap analysis before formal TISAX assessment
3. Rate each chapter against the 0–5 maturity scale
4. Identify "must" criteria that are below maturity 3 — these are mandatory gaps
5. Create corrective actions for all gaps rated below the required level
6. Track progress toward closure before engaging the Audit Provider

---

## Maturity Scale Reference

| Level | Name | Description |
|-------|------|-------------|
| 0 | Incomplete | Process/control does not exist or fails its basic purpose |
| 1 | Performed | Process informally performed; depends on individual know-how; not documented |
| 2 | Managed | Process planned, tracked, and documented; may not be consistently applied |
| 3 | Established | Defined process consistently applied across the organization; standard evidence exists |
| 4 | Predictable | Process quantitatively measured; performance tracked against metrics and KPIs |
| 5 | Optimizing | Continuous improvement process; root cause analysis and proactive enhancement |

**TISAX minimum requirement:** All "must" criteria must achieve Maturity **≥ 3**.
Standard criteria should achieve Maturity ≥ 3; minor shortfalls may be tolerated as
minor non-conformances at the Audit Provider's discretion.

## Corrective Action Plan Template

For each identified gap, create a corrective action record:

---

### Corrective Action Record

| Field | Detail |
|-------|--------|
| **Reference ID** | GAP-[Chapter]-[Sequential number] e.g. GAP-07-001 |
| **Chapter** | VDA ISA chapter number and name |
| **Assessment Area** | The specific requirement area |
| **Requirement Type** | Must / Standard |
| **Current Maturity** | Observed maturity level (0–5) |
| **Target Maturity** | Required level (typically 3) |
| **Gap Description** | Free text: what is missing, what failed, what is partially in place |
| **Root Cause** | Why does this gap exist? (no policy, no process, no evidence, inconsistent practice) |
| **Corrective Action** | Specific action(s) to close the gap |
| **Owner** | Name and role of person responsible for remediation |
| **Target Completion Date** | Realistic date by which gap will be closed |
| **Evidence of Closure** | What evidence will demonstrate the gap is closed? |
| **Status** | Open / In Progress / Closed / Verified |
| **Auditor Verification** | Date and name of person who verified closure |

## Evidence Collection Checklist

Compile the following before the formal Audit Provider engagement:

```
Before Assessment — Documentation Pack
[ ] Information security policy (current, approved, dated)
[ ] IS Manager / CISO role description or appointment
[ ] Asset register / CMDB (with classification)
[ ] Risk register (or reference to risk assessment)
[ ] All IS procedure documents (list by chapter)
[ ] Recent internal IS audit report (within 12 months)
[ ] Training records: IS awareness completion list (all staff)
[ ] Access review records (most recent periodic review)
[ ] Privileged access user list with rationale
[ ] ISMS scope document
[ ] Network architecture diagram
[ ] Completed self-assessment against VDA ISA (scored)
[ ] Corrective action plan with status

Before Assessment — Technical Evidence
[ ] Anti-malware management console screenshot (all endpoints, definitions current)
[ ] Vulnerability scan report (recent, within 3 months)
[ ] MFA deployment confirmation (VPN, admin interfaces)
[ ] Backup test records (most recent restorability test)
[ ] Patch management report (showing SLA compliance)
[ ] Encryption implementation evidence (BitLocker, TLS config)
[ ] User provisioning and deprovisioning procedure and sample records
[ ] Change management records for recent significant change
[ ] Security event log configuration and retention settings

Before Assessment — Supplier/Physical Evidence
[ ] Supplier contract template with IS clauses
[ ] Key supplier IS assessment records
[ ] Cloud service inventory with security assessment notes
[ ] Physical security access log sample
[ ] Clear desk policy observation records or audit

[If Prototype Label]
[ ] Prototype handling policy
[ ] Prototype register / asset tracking records
[ ] Prototype area access control records
[ ] No-photography evidence (signs, device control)
[ ] Prototype transport procedure and recent transport records
[ ] Prototype NDAs for all personnel with prototype access

[If Data Label]
[ ] ROPA (Records of Processing Activities)
[ ] Privacy notice(s)
[ ] DPA templates and signed processor agreements
[ ] DPIA examples for high-risk processing
[ ] Breach notification procedure
[ ] DSAR procedure and sample records
```
